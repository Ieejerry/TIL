# 스택(Stack)

</br>

<a href="https://ibb.co/jzmVxk9"><img src="https://i.ibb.co/JqVxbjW/Kakao-Talk-20230301-160751722.jpg" alt="Kakao-Talk-20230301-160751722" border="0"></a>

</br>

스택(Stack)은 데이터 구조의 하나이다.

</br>

<a href="https://ibb.co/PGPrQBc"><img src="https://i.ibb.co/hF4gfpB/Kakao-Talk-20230301-160751722-01.jpg" alt="Kakao-Talk-20230301-160751722-01" border="0"></a>

</br>

스택은 물건을 세로로 쌓는다고 생각하면 이해가 쉽다.

</br>

<a href="https://ibb.co/7QGTsKz"><img src="https://i.ibb.co/vhJr0kB/Kakao-Talk-20230301-160751722-02.jpg" alt="Kakao-Talk-20230301-160751722-02" border="0"></a>

</br>

쌓여있는 물건들 중에서 하나를 꺼내려면 위에서부터 차례로 꺼내야 한다.

</br>

<a href="https://ibb.co/0VvP5zg"><img src="https://i.ibb.co/Hp3mZ9W/Kakao-Talk-20230301-160751722-03.jpg" alt="Kakao-Talk-20230301-160751722-03" border="0"></a>

</br>

<a href="https://ibb.co/QN6JCdR"><img src="https://i.ibb.co/vPJcQYG/Kakao-Talk-20230301-160751722-04.jpg" alt="Kakao-Talk-20230301-160751722-04" border="0"></a>

</br>

스택에 데이터를 추가할 때는 가장 위에 추가된다.

</br>

<a href="https://ibb.co/D5DhvJP"><img src="https://i.ibb.co/3By2j35/Kakao-Talk-20230301-160751722-05.jpg" alt="Kakao-Talk-20230301-160751722-05" border="0"></a>

</br>

스택에 데이터를 추가하는 작업을 '푸쉬(push)'라고 한다.

</br>

<a href="https://ibb.co/McLnLxx"><img src="https://i.ibb.co/rxJkJRR/Kakao-Talk-20230301-160751722-06.jpg" alt="Kakao-Talk-20230301-160751722-06" border="0"></a>

</br>

<a href="https://ibb.co/VSWrBR9"><img src="https://i.ibb.co/85DT8qs/Kakao-Talk-20230301-160751722-07.jpg" alt="Kakao-Talk-20230301-160751722-07" border="0"></a>

</br>

스택에서 데이터를 꺼내는 경우 가장 위, 즉 가장 최근에 추가된 데이터부터 꺼낸다.

</br>

<a href="https://ibb.co/TBhBSnV"><img src="https://i.ibb.co/ccTcHpG/Kakao-Talk-20230301-160751722-08.jpg" alt="Kakao-Talk-20230301-160751722-08" border="0"></a>

</br>

스택에서 데이터를 꺼내는 작업을 '팝(pop)'이라고 한다.

**스택처럼 나중에 넣은 것을 먼저 꺼내는 구조를 'Last In First Out'이라고 하며 앞 글자만 따서 'LIFO'라고도 한다.**
